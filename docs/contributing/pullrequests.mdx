# How to Create a Pull Request (PR)

Firstly, we want to extend our gratitude for your interest in contributing to AgentGPT! Your efforts will greatly benefit the entire community.

Follow these steps to contribute:

1. **Fork the Repository**: Navigate to [AgentGPT on GitHub](https://github.com/reworkd/AgentGPT) and click on the `Fork` button in the upper right corner. This will create a copy of the repository in your own GitHub account.

2. **Clone the Repository**: Clone your forked repository to your local machine. This can be done by clicking on the `Code` button in your GitHub repository, copying the URL, and then opening your terminal and running the command `git clone <url>`.

3. **Create a New Branch**: Once inside the cloned repository on your local machine, use the command `git checkout -b your-branch-name` to create a new branch.

4. **Make Your Changes**: Now you're ready to make changes to the codebase. Please ensure you adhere to the project's code style and conventions.

5. **Commit Your Changes**: After making your changes, use the command `git add .` to stage your changes. Then, commit your changes using the command `git commit -m "commit message"`. Make sure your commit message is clear and descriptive.

6. **Push Your Changes**: Push your changes to your forked repository on GitHub using the command `git push origin your-branch-name`.

7. **Create a Pull Request**: Navigate back to your forked repository on GitHub and click on the `New pull request` button. Then, fill out the Pull Request template as detailed below.

## Pull Request Template

When submitting a pull request, please use the following template:

```markdown
Title: Brief Description of Changes

## Description

Provide a detailed description of the changes you have made, explain why you've made these changes, and how they will benefit the project.

## Styling

Elaborate on any specific styling changes you've made, including any modifications to the UI or design.

## Technical Decisions

Discuss any technical decisions you made during your development, such as why you chose one solution over another.

## Linked Issues

If applicable, provide links to any related issues that your changes address.

## Screenshots / Videos

If possible, include screenshots or videos of any visible changes to the application or the new functionality.
```

## What Happens Next?

- A team member will review your PR and leave comments if anything needs to be changed.
- Be prepared to revise your PR as necessary.
- Once everything looks good, your changes will be merged into the main branch!

We release weekly updates and always acknowledge the work of our contributors! It's a fantastic way to get your name out there while contributing to a good cause.

Please ensure you adhere to the [ESLint Style Guide](https://eslint.org/) when formatting your code.
